### Autonomous games need a deterministic inspection handoff state

- Context: A generated real-time prompt-game passed functional QA, but the visible handoff screen initially reflected a mid-QA state: a changed difficulty, elapsed best time, and an already spawned edge hazard. That state was valid gameplay, but it made UI alignment and Game Feel inspection noisier than necessary.
- Impact: Timer, physics, wave, spawn, and survival games can leave users inspecting an accidental test state instead of a clean representative state. This can make edge-spawn objects look clipped, make HUD counters appear inconsistent, and hide whether the default layout is actually polished.
- Trace: The correction added a dedicated inspection helper that resets the run, restores the default difficulty, clears hazards, pauses the simulation, and exposes a stable status message. Fresh QA returned `paused:true`, `difficulty:"Normal"`, `hazard_count:0`, `elapsed:0.0`, screenshot `nonblank:true`, `possible_clipping:false`, and no runtime errors.
- Improvement: For autonomous prompt games, keep separate helpers for stress testing and user inspection. After QA, hand off a deterministic, paused, representative state unless the user explicitly needs to inspect a failure or active-combat state.
- Status: Added as a reusable inspection and QA handoff rule.
