## 1. git에 없는 상태 (이 문서가 유일한 기록 — 중요)

- **Godot Asset Store**: v0.8.0이 2026-07-13 업로드됨, 현재 **Pending Approval**.
  Min `Godot 4.2` / Max `Godot 4.7`, Stable 체크. Settings 탭의 Summary/Description은
  upload doc의 블록과 바이트 단위 일치 검증 완료. **스토어 폼 조작은 사용자가
  직접 한다** — 에이전트는 준비/검증까지만. 승인되면 v0.8.0 관련 남은 일 없음.
- **Homebrew 탭은 별도 저장소** <https://github.com/NotNull92/homebrew-hera>
  (`Formula/hera.rb`, 0.8.0). 이 Windows PC에서는 brew를 실행할 수 없어 정적
  검증만 된 상태 — macOS/Linux 테스트는 미완.
- **winget (과거 상태)**: `packaging/winget/`의 매니페스트는 `winget validate`를
  통과했지만 공개 PR은 열리지 않았다. 이 계획은 2026-07-14에 폐기됐으므로 제출·갱신·
  홍보하지 않는다. 최신 결정은 [`2026-07-14-codex.md`](2026-07-14-codex.md)에 있다.
- **Scoop**: 이 PC에 scoop 미설치 → end-to-end `scoop install` 테스트는 못 했다.
  (해시·zip 구조·바이너리 실행은 실물 릴리스 자산으로 검증 완료.)
- **라이브 에디터**: 사용자가 Godot 4.7 에디터를 켜둔 채 작업한다. pid는 세션마다
  다르니 `go run . instances`로 발견하라. **광역 taskkill 절대 금지 — PID 지정
  kill만.** 사용자의 에디터를 격리 실험에 쓰지 말 것(격리 스모크는 임시 최소
  프로젝트 + 별도 headless 에디터로; `docs/SUPPORT_MATRIX.md`의 Method 참고).
- **Windows 함정**: `Godot_*_console.exe`는 래퍼이고 실제 에디터는 자식
  프로세스로 뜬다 → 하트비트가 **자식 pid**로 생성된다.
  `~/.hera-agent-godot/instances/`를 실행 전 스냅샷하고 새 fresh json을 기다려라.
- **로컬 Godot 4.7 바이너리** (PATH에 없음):
  `C:\Users\PC\Downloads\Godot_v4.7-stable_win64.exe\Godot_v4.7-stable_win64_console.exe`
- **`go test -race`는 이 PC에서 실패**한다(백신이 race 계측 바이너리를 차단).
  CI(ubuntu)가 race를 커버하므로 로컬은 race 없이 돌리고 그 사실을 보고하라.
- **릴리스 자산 이름은 버전 없는 고정 이름**(`hera-windows-amd64.zip` 등,
  `latest/download` 지원용). 패키지 매니페스트에는 반드시
  `/releases/download/v<ver>/…` 버전 고정 URL + 그 릴리스 `checksums.txt`의
  해시를 쓴다.

## 3. 작업 규칙 요약 (전체는 AGENTS.md)

- Go 게이트: `go build ./... && go vet ./... && go test -shuffle=on -count=1 ./...`
  + 변경 파일 `gofmt`. GDScript 게이트: 위 로컬 바이너리로
  `--headless --path . --check-only --script res://…`.
- CLI 표면(명령/플래그/JSON) 변경 시 문서 동기화: `cmd/root.go` 도움말,
  `AGENTS.md`, `docs/COMMANDS.md`, `README.md`, `README.ko.md`, 계약 변경이면
  `docs/CONTRACT.md` + 골든 갱신(`go test ./cmd -run TestContract -update`).
- 버전 bump/태그/릴리스는 **사용자가 요청할 때만**.
- 사용자 대화는 한국어, 커밋 메시지/저장소 문서는 영어.
- 커밋은 작게, 메시지는 서술적으로 — 상대 에이전트(Claude Code)는 git으로만
  맥락을 복원한다. 저장소 밖 상태가 바뀌면 `docs/`에 기록을 남겨라.
- 외부 공개 행동(서드파티 PR, 스토어 제출, 릴리스)은 실행 전 사용자 확인.
- 작업을 넘길 때는 `docs/handoff/<날짜>-<대상>.md`로 새 인수인계 문서를 남겨라.
