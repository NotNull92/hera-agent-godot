## Detailed Description

Paste verbatim into the Asset Store **Detailed Description** field (Markdown, 4885 / 20000 chars).

```markdown
**Hera Agent Godot** lets AI coding agents inspect and control a **live Godot 4.7+ editor** in real time — instead of guessing scene structure from stale training data, the agent acts on the *real* editor and checks the result.

This Asset Library entry is the **editor addon** (the in-editor bridge). The companion **`hera` CLI** lives on GitHub and is what the agent actually runs.

## Why a CLI, not MCP?

Godot already has a healthy MCP-addon ecosystem — Hera makes the opposite bet on purpose. MCP servers pay for breadth in tokens: dozens to 100+ tool schemas plus verbose JSON responses sit in the agent's context every turn.

Hera delivers **MCP-grade reach over the live editor as a compact-JSON-by-default CLI** — one command per action, minimal tokens, and it works with anything that can run a shell command (pipes, `batch`, CI, any agent), not just MCP clients. The agent loads **zero tool schemas**: the only resident surface is one small cacheable doc (~1k tokens) that stays flat no matter how many commands exist, versus an estimated ~4k–31k resident tokens for sampled Godot MCP servers.

## What the agent can do

- **Read the editor** — live status, scene tree, node property dumps, open scenes, editor selection, and ClassDB inspection.
- **Edit the scene** — add / set / remove nodes, attach & detach scripts, wire resources, connect & disconnect signals. Every write is a single undoable step (Ctrl+Z).
- **Scripts & project** — create scripts, make folders, list and scan project files, inspect and refresh resources and UIDs.
- **Run GDScript** — evaluate an expression against the edited scene through an `EditorInterface` proxy.
- **Drive a running game** — inspect and mutate the runtime tree, call helper methods, click controls semantically, inject real input and prove what was delivered, read the runtime UI tree, and assert on state.
- **Proof-first QA** — capture and analyze screenshots, run requirement-covered QA scenarios from a file, and get a one-shot runtime health verdict with `game qa diagnose`.
- **Diagnostics** — tail the Godot log with error/warning filters and summarize problems.

## New in v0.8.0 — a contract you can build on

- **Documented output contract** (`docs/CONTRACT.md`) — per-command response fields with stable/experimental tiers, exit codes (0 success / 1 runtime failure / 2 usage error), and data-only stdout, so scripts and agent harnesses can depend on Hera's output.
- **Golden contract tests in CI** — the stdout of stable commands is pinned byte-for-byte end-to-end; an accidental breaking change fails the build.
- **Opt-in shared-token auth** — lock the localhost bridge to clients that know a secret (`~/.hera-agent-godot/token` or `HERA_AGENT_GODOT_TOKEN`), with a documented threat model (`docs/SECURITY.md`). Still `127.0.0.1`-only.
- **Verified support matrix** — the addon loads and answers the CLI on every stable from **Godot 4.2 to 4.7** (spot-checked; 4.7 fully QA'd). CI runs the GDScript gate on both ends.
- **`game qa diagnose`** — a generic runtime health verdict (editor diagnostics, game process count, runtime tree/UI reads, screenshot analysis) whose exit code mirrors the result.
- **`--timeout <ms>` global flag** and one unified name: the CLI installs as **`hera`** (`hera-agent-godot` stays as a transitional alias).
- Cleaner startup in user projects: the Hera panel no longer error-spams about assets that only exist in the dev repo.

## Command surface

`status`, `instances`, `run` / `stop`, `scene`, `editor`, `script`, `project`, `classdb`, `node` (read + write + resource/script wiring), `signal`, `resource`, `game` (runtime inspect + input + input-log + set/call/click + assert + QA + screenshot), `guidance`, `game_feel`, `output`, `diagnostics`, `eval`, `screenshot`, `batch`, and `smoke` — with `--json` (pretty) and `--ids` (paths only) output modes plus `--instance` and `--timeout` global flags; compact JSON is the default.

## How it works

The addon is an `@tool` `EditorPlugin` that runs a localhost HTTP server and advertises the editor to the CLI via `~/.hera-agent-godot/instances/`. The Go CLI discovers the editor and sends one compact JSON request per command, executed on the editor main thread through `EditorInterface`. No cloud and no account — everything stays on localhost, optionally behind the shared token.

## Install

1. Use a **Godot 4.7+** build (recommended; verified to load and answer the CLI down to **4.2** — see `docs/SUPPORT_MATRIX.md` in the repo).
2. Copy `hera_agent_godot/` into your project's `res://addons/`.
3. Enable **Project → Project Settings → Plugins → Hera Agent Godot**.
4. Grab the `hera` CLI from GitHub (one-line installer or `go build`) and start driving the editor from your agent.
```

## Changelog

- Added `docs/CONTRACT.md`, the output contract for the `hera` CLI: public/internal boundary, stability tiers per command (stable vs experimental), output streams and modes, JSON conventions, exit-code semantics (0/1/2), and verdict-command behavior.
- Added golden contract tests (`cmd/contract_golden_test.go` + `cmd/testdata/contract/`) that run the real CLI end-to-end against a mock editor and byte-compare stdout with goldens captured from a live Godot 4.7 editor; they run in CI, so accidental contract breaks fail the build.
- Implemented the `--timeout <ms>` global flag (per-request HTTP timeout, default 5000 ms; `--timeout N` or `--timeout=N`; invalid values exit 2). Previously documented but unimplemented.
- Usage errors (unknown command, bad flags) now print to stderr, keeping stdout data-only per the contract.
- Added `game qa diagnose`: a generic, project-independent runtime health verdict over editor diagnostics, live game process count, runtime tree/UI reads, and screenshot analysis, with `--lines`, `--max-errors`, `--max-warnings`, and `--path` thresholds and an exit code that mirrors `ok`.
- Added `docs/SECURITY.md` (localhost threat model) and opt-in shared-token auth: when `~/.hera-agent-godot/token` or `HERA_AGENT_GODOT_TOKEN` is set, `/rpc` requires a matching `X-Hera-Token` header; mismatches get 401 and the CLI exits 1 with `unauthorized: ...`. The editor log and Hera panel show `token auth on`.
- Published `docs/SUPPORT_MATRIX.md`: verified minimum Godot 4.2 — every stable from 4.2 to 4.6 passes `--check-only` over all addon scripts, loads the plugin headless, and answers `status` over the live bridge; 4.7 remains the fully-QA'd baseline. The CI GDScript gate now runs on both 4.2-stable and 4.7-stable.
- Unified the invocation name on `hera` across CLI help, docs, and installers; `hera-agent-godot` remains a transitional alias (symlink from `install.sh`, `.cmd` shim from `install.ps1`).
- The Hera main-screen panel no longer error-spams on editor start in user projects: the logo and display-font loaders probe before loading and fall back silently.
- Expanded the roadmap with the standardization arc (Phases 7–9: contract & trust → reach & agent-side distribution → v1.0 standard declaration) and added a generic runtime-diagnosis step to the prompt-game workflow guide.
