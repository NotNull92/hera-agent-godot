**Hera Agent Godot** lets AI coding agents inspect and control a **live Godot

This Asset Store entry contains the editor addon. The companion **`hera` CLI**

installers in the repository.

## Why Hera

Hera exposes the real editor through compact shell commands. Agents load no
large tool-schema catalog: they call one command, receive compact JSON, and
verify the result through the editor, runtime, diagnostics, or screenshots.
Everything stays on localhost, with optional shared-token authentication.

## What agents can do

- Inspect the live editor, open scenes, selection, scripts, resources, Theme
  items, and Godot ClassDB data.
- Add, update, instance, and remove scene nodes with editor undo support.
- Create and attach scripts, wire signals, and manage project resources.
- Run scenes and inspect or control the live game tree.
- Click visible controls semantically, inject input, and verify delivery.
- Capture and analyze screenshots or compare two captures locally.
- Execute requirement-covered QA scenarios and summarize diagnostics.
- Apply bundled Godot-native UI theme QA guidance for spacing, type, color,
  contrast, containers, and decoration.

## New in v1.0.0

- The documented stable CLI contract is frozen for the v1 major line, backed
  by contract goldens and a SemVer/deprecation policy.
- `theme get/set` reads and updates Godot `Theme` resource items that ordinary
  property mutation cannot reach.
- `screenshot diff` reports changed pixels, ratio, and bounding box locally.
- Discovery retries heartbeat replacement races over bounded delays.
- Diagnostics report logging blind spots instead of returning false-clean
  results, and expected runtime file-exchange races stay quiet.
- A migration guide documents the compatible v0.9-to-v1 upgrade path.

## Upgrade

Upgrade the CLI and addon together, fully restart Godot, then run
`hera status` and `hera smoke --skip-game`. Existing scenes, opt-in auth, and
the transitional `hera-agent-godot` command alias remain compatible.

## Changelog

### Stable v1 contract and upgrade path

- Froze the documented **stable** CLI surface for the v1 major line: command
  invocation, output streams, exit-code meanings, response field names, and
  JSON types now carry a compatibility promise backed by contract goldens.
- Adopted semantic versioning and a written deprecation policy. Stable fields
  may grow additively, but removal or incompatible change requires deprecation
  in a minor release and waits until the next major release.
- Added a v0.9-to-v1 migration guide. The upgrade is intentionally compatible:
  update the CLI and addon together, fully restart Godot, and keep existing
  scenes, scripts, shared-token settings, discovery paths, and the transitional
  `hera-agent-godot` command alias.
- Kept the new `theme` and `screenshot diff` responses explicitly
  **experimental** so their shapes can mature without weakening the stable v1
  contract.

### Theme resources and visual QA

- Added `theme get <res://theme.tres> [--type <ThemeType>]` to read colors,
  constants, and font sizes from Godot `Theme` per-type maps that ordinary
  resource property inspection cannot reach.
- Added `theme set` with repeatable `--color`, `--constant`, and `--font-size`
  items. It saves the Theme resource, reports `undoable:false`, rejects unknown
  theme types and empty writes, and prints colors to six decimals so float32
  values round-trip without noisy tails.
- Added local `screenshot diff <before> <after> [--threshold N]`. It works
  without an editor and reports dimensions, changed-pixel count and ratio,
  maximum channel delta, identity, and the exact changed bounding box.
- Made screenshot comparison tolerant of anti-aliasing wobble with a per-channel
  threshold while still reporting `max_delta` below that threshold; mismatched
  dimensions and unreadable or invalid images fail instead of producing a
  misleading comparison.
- Shipped the Godot-native `ui-theme-qa` skill in both Claude Code and Codex
  plugin trees. It measures six areas: spacing, type scale, color convergence,
  WCAG contrast, container structure, and decoration.
- The four mechanical areas update undoable per-node `theme_override_*` values
  in dependency order, then re-read live predicates and confirm the rendered
  result. Container and decoration findings remain report-only because deleting
  or re-parenting nodes can break semantic dividers, brand marks, tooltips, and
  script node paths.
- Rooted the reference corpus in Godot's own default theme: 4 px spacing,
  16/20/24/28 font anchors, and the engine's 1.25 heading step. WCAG remains the
  accessibility authority; no outside color ramp is imported over a project's
  visual language.
- Added safe color convergence that chooses the project's most-used existing
  value for near-duplicates, and contrast repair that preserves hue, computes
  both reachable lightness bounds, and reports when neither bound is valid.
- Fixed real mixed-UI inspection failures by reading only the theme tokens each
  Control class actually defines, and fixed type-scale collision handling so a
  hierarchy correction does not shrink unrelated peer controls.
- Deliberately excluded automatic wholesale restyling. Hera can apply a chosen
  style through `node set`, `theme set`, and `resource create Theme`, but the QA
  workflow does not invent taste or silently replace a project's design.

### Live-editor reliability and truthful diagnostics

- Hardened Windows editor discovery against the non-atomic heartbeat rename
  window. Empty scans now retry over growing 25/50/100/200 ms delays and stop
  at the first live instance; the final QA sweep passed 39/39 discovery calls
  and 10/10 repeated `editor state` calls.
- Reduced heartbeat warning noise without hiding real failures: a single
  publish collision is treated as expected, while eight consecutive failures
  still emit one warning before the five-second freshness window is at risk.
- Replaced noisy `JSON.parse_string()` calls with quiet, checked parsing at the
  runtime request, response, game-instance, and Game Feel line-reader polling
  boundaries. Mid-write files now retry without printing false engine errors;
  malformed HTTP request bodies remain real protocol errors.
- Made `diagnostics` and `output` read Godot's effective feature-tagged logging
  setting and path. An unreadable log now returns `available:false` with a hint,
  and diagnostics returns `clean:false` instead of claiming a project is clean
  from missing evidence.
- Updated `game qa diagnose` to fail the editor-diagnostics check when the log
  is explicitly unavailable, while preserving compatibility with older addons
  that do not return the new `available` field.
- Corrected the documented visibility boundary: normal file logging covers the
  running project, not the editor console. Launching the editor with
  `--log-file <path>` is the supported way to capture editor output.
- Documented that disabling and re-enabling the plugin does not reload preloaded
  addon scripts; a full editor restart is required before a live smoke can prove
  addon source changes.

### Release integrity and documentation

- Added the missing tracked UID sidecar for `theme_tool.gd`, preventing Godot
  from regenerating different script UIDs across clones.
- Synchronized English/Korean command documentation, the architecture and
  output contracts, the UI Theme QA design record, support evidence, release
  baseline, and Asset Store form copy for the v1 declaration.

## Manual Store Step

1. Review and commit the release-prep changes, create annotated tag `v1.0.0`,
   and push only with explicit owner approval.
2. Wait for `.github/workflows/release.yml`; it must pass the race test and
   publish the CLI archives, addon ZIP, and `checksums.txt`.
3. Download the published addon ZIP, verify its SHA256 against
   `checksums.txt`, and update the Release SHA line above. Prefer that asset
   over the local ZIP for Store submission.
4. On the Asset Store **Settings** tab, replace the stale public Summary and
   Description with the blocks above and save.
5. On the **Versions** tab, upload the verified Release ZIP as `v1.0.0`, Stable,
   minimum Godot 4.2, maximum blank, with the Changelog and Additional
   Information above. The owner performs the final submission.
6. After the GitHub Release exists, update npm, Homebrew, and Scoop using the
   published checksums; the one-line installers follow the latest release
   automatically.
