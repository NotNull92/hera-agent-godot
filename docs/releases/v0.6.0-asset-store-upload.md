## Summary

Hera Agent Godot gives AI coding agents eyes, hands, and proof inside a live Godot editor through shell-native, low-token commands for inspection, editing, runtime control, screenshots, diagnostics, and QA.

## Description

Hera Agent Godot is a Godot 4.7+ editor addon plus companion CLI for agent-driven development. It runs inside the real editor, communicates over localhost, and exposes compact commands for live editor status, scene/node inspection, editor selection, script and resource workflows, signal wiring, runtime game inspection, semantic UI clicks, screenshots, diagnostics, and automated QA scenarios.

This release strengthens Hera's proof-first workflow: agents can narrow large UI reads, discover deterministic runtime QA helpers, and fail QA scenarios when declared prompt requirements are not covered by executable checks.

## Changelog

- Added scoped runtime UI reads with `game ui tree --path`, `--depth`, `--fields`, `--type`, and `--text` to reduce token usage on Control-heavy scenes.
- Added focused editor reads with `node get --prop` and `node get --props`.
- Added `game qa discover [path]` so agents can list callable runtime `qa_*` helpers before invoking deterministic QA methods.
- Added object-format `game qa --file` scenarios with top-level `requirements` and per-step `covers`; scenarios now fail when declared requirements are not covered.
- Added `docs/GAME_PROMPT_WORKFLOW.md` for prompt-driven game implementation with requirement-led QA.
- Added `docs/IDENTITY.md` and aligned README/addon README language around live editor truth, low-token control, and proof-first QA.
- Documented v0.6.0 analysis findings in `docs/reports/v0.6.0.md`.
