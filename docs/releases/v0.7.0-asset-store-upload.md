## Summary

Hera Agent Godot gives AI coding agents eyes, hands, and proof inside a live Godot editor through shell-native, low-token commands for inspection, editing, runtime control, screenshots, diagnostics, and QA.

## Description

Hera Agent Godot is a Godot 4.7+ editor addon plus companion CLI for agent-driven development. It runs inside the real editor, communicates over localhost, and exposes compact commands for live editor status, scene/node inspection, editor selection, script and resource workflows, signal wiring, runtime game inspection, semantic UI clicks, screenshots, diagnostics, and automated QA scenarios.

## Changelog

- Added a dedicated `HeraAgent` editor main-screen panel backed by shared Hera settings. The panel now exposes live bridge status, Hera branding, and the persistent Game Feel Mode controls agents use before starting gameplay or UI work.
- Added a bundled `game_feel` knowledge base and CLI/tool surface. Agents can query concrete topics such as control feel, screen shake, hit stop, camera, sound, particles, reward feedback, UI bars, accessibility, and ethical feedback instead of relying on vague "juice" prompts.
- Added gameplay-wide `guidance game-feel` output alongside existing UI guidance. The response now separates gameplay feel from Control layout work and returns compact QA patterns for runtime verification, accessibility options, Honest Juice, and observable feedback evidence.
- Added runtime input injection with `game input`: mouse press/release/click/move, keyboard press/release, InputMap action press/release with strength, text input, modifiers, physical key routing, and viewport routing for focused Control input.
- Added `game input-log` diagnostics so QA can prove which runtime inputs were actually delivered. The log tracks click coordinates, button state, short/long press classification, key names, modifiers, active keys, and active mouse buttons.
- Expanded `game qa --file` scenarios with `game.input` and `game.input_log` steps, letting requirement-covered QA drive primary keyboard, mouse, touch-style, and controller/action paths instead of validating only helper-only code paths.
- Strengthened the runtime inspector and viewport action bridge so live game QA can combine semantic clicks, injected input, deterministic helper calls, runtime UI reads, screenshots, and diagnostics in one ordered proof flow.
- Added Game Feel-aware editor hints: when Game Feel Mode is enabled, relevant `node add` operations can return compact `agent_hint` topics that point agents toward the matching `game_feel` guidance.
- Promoted v0.7.0 prompt-game QA findings into reusable guidance: ordered state-changing QA, direct affected-scene parsing when runtime registration is empty, current-project helper discovery, hidden-state precondition resets, deterministic helper quiescence, and stable inspection handoff states.
- Added visual/layout QA rules from hands-on prompt-game cycles: bounded child visuals for pieces/markers/tokens, derived grid insets, stable interactive frames with animated child content, live viewport-sized playfields, readable contrast, and parent/child rect verification through runtime UI tree reads.
- Added Game Feel verification rules from hands-on testing: require perceivable multi-channel feedback, expose target/channel/duration/intensity state, verify screen shake or board motion at the intended scope, and avoid treating feature-list strings or booleans as proof of player-visible feel.
- Added route/path QA rules for games with lanes, rails, patrols, projectiles, or tower-defense paths: drawing and movement should share one smoothed authoritative geometry, and screenshots plus live movement steps should prove entities stay aligned with the rendered path.
- Expanded the GDScript agent guide with typed-copy helpers for `Dictionary`/`Variant` arrays, style-resource separation from frame loops, selector/label synchronization after programmatic state changes, and direct runtime helper path discovery from the installed addon.
- Added `docs/reports/v0.7.0.md`, capturing the prompt-game QA cycles that produced these reusable rules, including UI consistency, keyboard input, Game Feel salience, screen shake scope, path smoothing, and runtime helper reliability findings.
- Refreshed the English, Korean, and addon README files so the public documentation matches the current v0.6.0 release baseline while v0.7.0 adds the new Game Feel and runtime input QA surfaces.
